# CHANGELOG ExportEPUB

## 0.1.22-beta

- Улучшено качество конвертации FB2 → EPUB без изменения тестовой инфраструктуры:
  - `empty-line` теперь выводится как безопасный блочный разделитель `<div class="empty-line">`, а не как искусственный абзац.
  - `subtitle` стал контекстным: внутри стихов выводится как строка стихотворного блока, в заголовках — как заголовочный подзаголовок, в обычном тексте — как отдельный подзаголовок.
  - Аннотация на отдельной странице оборачивается в `section.annotation`, при этом сохраняется внутренняя FB2-структура.
  - Улучшены CSS-правила для стихов, эпиграфов, цитат, аннотаций, таблиц, сносок, пустых строк и кода.
  - В обычных главах больше не дублируется верхний `h1`, если в тексте уже есть заголовок из FB2 `title`.
  - Пустые ячейки таблиц получают `&#160;`, чтобы не схлопываться в читалках.
  - `code` получает `white-space: pre-wrap` и устойчивое моноширинное оформление.
  - `binary` без `content-type` или с неправильным `content-type` теперь может быть принят, если реальный тип изображения определяется по сигнатуре.
  - Для секций без явного `title` в навигации используется первый короткий абзац, затем fallback `Глава N`.
- Исправлен `BUILD_BATCH_WIN32_RELEASE.cmd`: теперь он явно собирает сначала `ExportEPUB.dll`, затем `ExportEPUBBatch.exe`, проверяет наличие обоих файлов и показывает отдельные логи сборки.

## 0.1.21-beta

- Добавлены практические режимы массового экспорта без изменения ядра EPUB:
  - `--filename source|title|author-title|title-author|series-number-title` — выбор правила имени выходного EPUB.
  - `--skip-existing` — явный режим пропуска уже существующих EPUB.
  - `--newer-only` — пересоздавать только отсутствующие или устаревшие EPUB.
- Для `--filename` добавлено чтение `book-title`, первого автора и первой `sequence` из FB2.
- Имена файлов очищаются от запрещённых символов Windows, обрезаются до безопасной длины и получают fallback на имя исходного FB2.
- Добавлена защита от дублей выходных имён внутри одного запуска: `_2`, `_3`, `_4`.
- CSV-отчёт дополнен полем `SkipReason`.
- Код формирования XHTML/OPF/ZIP не менялся.

## 0.1.20-beta

- Добавлены практические возможности массовой конвертации:
  - `ExportEPUBBatch.exe --from-list <file>` — конвертация только файлов из UTF-8 списка.
  - `ExportEPUBBatch.exe --check-input` / `--list` — быстрый просмотр найденных FB2 без конвертации.
  - CSV-отчёт расширен полями `ExitCode`, `DurationMs`, `SourceSize`, `OutputSize`.
- `TEST_EXPORT_PROBLEM_FILES.cmd` переведён на общий механизм `--from-list`.
- Добавлен редактируемый список `PROBLEM_FILES_ExportEPUB.txt` для повторной проверки выбранных книг.
- Код экспорта EPUB не менялся.


## 0.1.19-beta

- Добавлен TEST_EXPORT_PROBLEM_FILES.cmd для повторной проверки только ранее проблемных FB2-файлов.
- Командный файл конвертирует выбранные FB2 в EPUB 2/EPUB 3, запускает EPUBCheck и собирает небольшой ZIP-архив для анализа.

# 0.1.18-beta

- Исправлены оставшиеся ошибки EPUBCheck на повторном большом тестовом наборе после 0.1.17-beta.
- Исправлен XHTML вида <h3><p class="image">...</p></h3>, который возникал из FB2 <subtitle><image .../></subtitle>. Теперь subtitle/title с блочным содержимым выводятся через безопасный <div class="subtitle/title">.
- В заголовках глав теперь сохраняются id, заданные на <title><p id="...">...</p></title>. Это исправляет внутренние ссылки вида #BdToc_... в книге «Хэлперн. Коллайдер».

- В CSS добавлены базовые правила для .title и .subtitle при блочном выводе заголовков.

- image/jpg нормализуется в image/jpeg для совместимости с EPUBCheck.
- Fallback identifier теперь создаётся как валидный urn:uuid:xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx.
