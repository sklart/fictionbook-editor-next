ExportEPUB: тестирование на большом объёме FB2
================================================

Основной командный файл:

    TEST_EXPORT_LARGE_FB2.cmd

Он предназначен для больших библиотек FB2: тысячи файлов, несколько гигабайт.
В отличие от TEST_EXPORT_AND_PACK.cmd он не упаковывает весь набор целиком,
а создаёт компактный архив с логами и только проблемными файлами.

1. Что настроить в TEST_EXPORT_LARGE_FB2.cmd
-------------------------------------------

В начале файла задаются переменные:

    set "FB2_INPUT_DIR=D:\fb2_test_suite"
    set "TEST_OUTPUT_ROOT=D:\fb2_large_test_out"
    set "ARCHIVE_PATH=D:\fb2_large_test_problems.zip"
    set "EPUBCHECK_JAR=D:\Tools\epubcheck.jar"

Для большого прогона рекомендуется оставить:

    set "NEWER_ONLY=1"
    set "OVERWRITE=0"
    set "FILENAME_MODE=source"

NEWER_ONLY=1 позволяет безопасно продолжать прерванный тест:
уже созданные и актуальные EPUB будут пропущены.

2. Запуск
---------

    cd /d D:\Download\FBeditor\src\export-epub
    BUILD_BATCH_WIN32_RELEASE.cmd
    TEST_EXPORT_LARGE_FB2.cmd

3. Частичный прогон по диапазону
--------------------------------

Для очень больших библиотек можно гонять частями:

    set "START_INDEX=1"
    set "MAX_FILES=1000"

Потом:

    set "START_INDEX=1001"
    set "MAX_FILES=1000"

И так далее. Нумерация START_INDEX начинается с 1 после сортировки списка FB2.

4. Что попадает в ZIP
--------------------

По умолчанию:

    D:\fb2_large_test_problems.zip

В архив попадают:

    logs\
    fb2\      только проблемные исходные FB2
    epub2\    только проблемные EPUB 2
    epub3\    только проблемные EPUB 3
    notes.txt

Если ошибок нет, архив будет содержать в основном логи и notes.txt.
Такой архив всё равно полезен: по нему можно подтвердить, что большой прогон прошёл чисто.

5. Новые параметры ExportEPUBBatch.exe для больших прогонов
----------------------------------------------------------

    --newer-only
        Пересоздаёт только отсутствующие или устаревшие EPUB.

    --start-index N
        Начать с N-го FB2 после сортировки.

    --max-files N
        Обработать не более N исходных FB2.

    --progress-every N
        Печатать прогресс каждые N запланированных экспортов.

    --keep-failed-output
        Не удалять частично созданный EPUB после ошибки.
        Обычно не нужно. По умолчанию битые/пустые выходные файлы удаляются.

6. Рекомендуемый порядок большого теста
--------------------------------------

Сначала короткая проверка:

    set "START_INDEX=1"
    set "MAX_FILES=50"

Если всё нормально, полный прогон:

    set "START_INDEX=1"
    set "MAX_FILES=0"

После завершения отправить архив:

    D:\fb2_large_test_problems.zip
