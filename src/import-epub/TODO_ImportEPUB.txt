Отложено до окончания тестирования большого EPUB-корпуса
---------------------------------------------------------
- MathML/formula import backend. Candidate pipeline:
  [MathML] -> mml2tex XSLT/XProc -> [LaTeX] -> MicroTeX -> [SVG/bitmap] -> LunaSVG/GDI+ -> [PNG] -> FB2 <image>/<binary>.
- Реализовывать только как optional backend/helper, не как обязательную зависимость ImportEPUB.dll.
- Предпочтительная архитектура:
  ImportEPUBMathHelper.exe или ImportEPUBMath.dll рядом с ImportEPUB.dll/ImportEPUBBatch.exe.
  Если helper/backend отсутствует или падает, основной импорт EPUB не должен ломаться.
- Этап 1:
  найти MathML в XHTML;
  не терять формулы;
  добавить настройки "Формулы MathML";
  добавить статистику MathMLFound/MathMLSkipped/MathMLErrors;
  fallback: plain text или LaTeX/text placeholder.
- Этап 2:
  подключить mml2tex через Saxon/SaxonC/Java-helper;
  сохранять полученный LaTeX в лог/diagnostics;
  добавить NormalizeLatexForMicroTeX().
- Этап 3:
  MicroTeX -> SVG;
  SVG -> PNG через существующий LunaSVG backend;
  вставка PNG в FB2;
  кеширование формул по hash(normalized MathML + size + scale + color);
  расширение CSV/HTML отчёта колонками MathMLFound, MathMLRendered, MathMLAsLatex, MathMLFallback, MathMLErrors, MathBackend.
- Основные риски:
  mml2tex требует XSLT/XPath 2+/Saxon, MSXML не подходит;
  MicroTeX может поддерживать не весь LaTeX, который выдаст mml2tex;
  inline-формулы в FB2/FBE требуют отдельной проверки;
  нужна защита по timeout/размеру/количеству формул на книгу.
