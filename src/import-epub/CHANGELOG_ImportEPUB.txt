- Improved EPUB notes detection for Standard Ebooks style endnotes stored as <li id="..."> inside an endnotes/footnotes container. These list-item notes are now collected into FB2 body name="notes" and noteref links can become type="note" links instead of ordinary unresolved internal links.

﻿fix70 - crash-isolated batch conversion and resume-report checks
----------------------------------------------------------------
- Added ImportEPUBBatch.exe --isolate-crashes for batch mode. Each EPUB is converted in a child ImportEPUBBatch process, so a fail-fast crash such as 0xC0000409 is written as FAILED_CRASH and the parent batch continues with the next file.
- Child conversion writes a temporary one-row CSV report that the parent merges into the main CSV report, preserving normal OK/OK_WITH_WARNINGS/FAILED statistics for non-crashing files.
- RUN_IMPORT_REGRESSION_TESTS.cmd and BIG_VOLUME_TEST default options now include --isolate-crashes.
- RUN_IMPORT_REGRESSION_TESTS.ps1 no longer treats SKIPPED rows in resume mode as failed semantic checks; skipped linear-algebra/friedrich rows are reported as warnings instead of false errors.
- Version bumped to 1.0.70.0.

fix69 - malformed HTML fallback and crash-resume regression run
----------------------------------------------------------------
- EPUB HTML/XHTML fallback now normalizes common HTML void elements (<link>, <meta>, <br>, <img>, etc.) into XML-safe self-closing tags before retrying MSXML loadXML.
- XML fallback now normalizes XML 1.1 declarations to XML 1.0 and removes invalid control characters, which helps OPF files rejected by MSXML with "bad version".
- RUN_IMPORT_REGRESSION_TESTS.cmd now has RESUME_MODE=1 by default and uses --skip-existing, so after a crash/interruption the run can continue without reconverting already written FB2 files.
- Fixed RUN_IMPORT_REGRESSION_TESTS.ps1 Find-Row single-result unwrapping, which could produce false "test file not found" warnings.
- For a completely clean regression run set RESUME_MODE=0.
