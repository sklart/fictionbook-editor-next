ExportDOCX — плагин экспорта DOCX для FictionBook Editor
=============================================================

Назначение
----------
ExportDOCX добавляет в FictionBook Editor экспорт книги FB2 в формат Microsoft Word DOCX.

Плагин рассчитан на 32-битный FB Editor, поэтому основная рабочая конфигурация сборки:

    Release | Win32

После сборки основной файл плагина:

    out\Release\ExportDOCX.dll

Дополнительно собирается консольная утилита массового экспорта:

    out\Release\ExportDOCXBatch.exe

Для ExportDOCXBatch.exe добавлены собственные ресурсы: иконка из res\ExportDOCX.ico и VERSIONINFO.
Справка, выводимая самой утилитой по -Help или /?, приведена на английском языке.

Возможности плагина
-------------------
Плагин поддерживает:

- экспорт текста FB2 в DOCX;
- экспорт обложки и изображений из блоков binary;
- ограничение максимальной ширины изображений;
- создание титульной страницы;
- перенос названия, авторов, жанров, серии, даты, аннотации и технических сведений FB2;
- сноски Word внизу страницы;
- концевые примечания Word;
- отдельный раздел «Примечания» в конце документа;
- автоматическое оглавление Word;
- выбор глубины оглавления от 1 до 4;
- закладки Word для section/@id;
- внешние и внутренние гиперссылки;
- обработку битых внутренних ссылок без создания повреждённых Word-гиперссылок;
- таблицы FB2 с базовым оформлением;
- стили для заголовков, цитат, эпиграфов, стихов, подписей к иллюстрациям, таблиц и кода;
- выбор профиля оформления;
- выбор размера страницы A4, A5 или Letter;
- выбор системного шрифта из списка установленных шрифтов Windows;
- настройку размера основного текста;
- автоматическое определение языка документа из FB2;
- автоматические переносы Word;
- верхний колонтитул с названием книги;
- номера страниц;
- отдельные секции Word для титульной/оглавления и основного текста;
- возможность не нумеровать титульную страницу и оглавление;
- возможность начать нумерацию основного текста с 1;
- проверку структуры DOCX после экспорта;
- диагностический TXT-отчёт рядом с DOCX;
- автоматическое открытие DOCX после экспорта;
- массовый экспорт FB2 через ExportDOCXBatch.exe.


Сборка
------
Для обычной сборки DLL и batch-утилиты запустите:

    BUILD_WIN32_RELEASE.cmd

После успешной сборки должны появиться:

    out\Release\ExportDOCX.dll
    out\Release\ExportDOCXBatch.exe

Для сборки только batch-утилиты можно использовать:

    BUILD_BATCH_WIN32_RELEASE.cmd

Файл BUILD_BATCH_WIN32_RELEASE.cmd собирает ExportDOCXBatch.exe вместе с зависимостью ExportDOCX.dll.


Регистрация DLL
---------------
FB Editor использует 32-битный COM-плагин. Даже на 64-битной Windows DLL нужно регистрировать через 32-битный regsvr32:

    C:\Windows\SysWOW64\regsvr32.exe

Для регистрации собранной DLL запустите:

    REGISTER_EXPORTDOCX.cmd

Скрипт ищет DLL в типовых местах:

    out\Release\ExportDOCX.dll
    out\Win32\Release\ExportDOCX.dll
    ExportDOCX.dll

Для сборки и регистрации одной командой используйте:

    REBUILD_REGISTER_WIN32_RELEASE.cmd

Для удаления регистрации:

    UNREGISTER_EXPORTDOCX.cmd

Важно: C:\Windows\System32\regsvr32.exe — это 64-битная версия regsvr32. Для 32-битного FB Editor она не подходит.


Очистка после сборки
--------------------
Для удаления результатов сборки и временных файлов запустите:

    CLEAN_AFTER_BUILD.cmd

Обычно удаляются каталоги:

    build\
    out\

а также промежуточные файлы Visual Studio, если они появились в папке проекта.


Как экспортировать из FB Editor
-------------------------------
1. Соберите проект в конфигурации Release | Win32.
2. Зарегистрируйте ExportDOCX.dll через REGISTER_EXPORTDOCX.cmd.
3. Запустите FB Editor.
4. Откройте FB2-книгу.
5. Выберите экспорт в DOCX.
6. В окне сохранения при необходимости нажмите «Настройки экспорта DOCX...».
7. Укажите имя DOCX-файла и сохраните документ.


Настройки экспорта
------------------
Окно настроек разбито на вкладки.


1. Основное
~~~~~~~~~~~
Экспортировать изображения
    Включает перенос изображений FB2 в DOCX. Используются ссылки image и соответствующие блоки binary.

Экспортировать обложку
    Добавляет изображение из description/title-info/coverpage. Если включена титульная страница, обложка используется в начале документа.

Ограничивать ширину изображений
    Масштабирует слишком широкие изображения, чтобы они не выходили за рабочую область страницы.

Макс. ширина, см
    Максимальная ширина изображения в сантиметрах. Для A4 обычно удобно 12–16 см.

Создавать титульную страницу
    Добавляет титульную страницу перед основным текстом. Используются данные из FB2: название, авторы, обложка и выбранные дополнительные сведения.

Включать аннотацию на титульную
    Добавляет annotation из description/title-info на титульную страницу.

Включать жанры
    Добавляет genre из title-info.

Включать серию
    Добавляет сведения sequence name/number, если они есть в FB2.

Включать сведения FB2
    Добавляет технические сведения из document-info: id, version, program-used и другие доступные поля.

Добавлять оглавление Word
    Создаёт автоматическое поле TOC. После открытия DOCX в Word может потребоваться обновить поле оглавления клавишей F9.

Глубина оглавления
    Максимальный уровень заголовков, попадающих в оглавление: 1, 2, 3 или 4.

Создавать закладки для разделов FB2
    Для section с атрибутом id создаются закладки Word. Они нужны для внутренних переходов.

Преобразовывать гиперссылки
    Преобразует внешние ссылки http/https/mailto и внутренние ссылки #id в гиперссылки Word. Битые внутренние ссылки не создаются как Word-гиперссылки и фиксируются в диагностике.


2. Примечания
~~~~~~~~~~~~~
Режим примечаний
    Сноски Word внизу страницы
        Примечания FB2 преобразуются в обычные сноски Word возле места ссылки.

    Концевые примечания Word
        Примечания собираются в стандартные endnotes Word.

    Раздел «Примечания» в конце документа
        body name="notes" добавляется отдельным разделом в конце DOCX.


3. Оформление
~~~~~~~~~~~~~
Выравнивать основной текст по ширине
    Выравнивает обычные абзацы по ширине. Не применяется к заголовкам, стихам, изображениям и служебным стилям.

Делать отступ первой строки
    Добавляет книжный отступ первой строки для обычных абзацев.

Начинать главы с новой страницы
    Перед главами и крупными разделами вставляется разрыв страницы.

Улучшать стили FB2
    Использует отдельные стили Word для стихов, цитат, эпиграфов, авторов цитат, подписей к иллюстрациям, таблиц и блоков кода.

Профиль оформления
    Книжный
        Times New Roman 12 пт, книжное оформление, выравнивание по ширине, отступ первой строки.

    Компактный
        Times New Roman 10,5 пт, уменьшенные поля и более плотная вёрстка.

    Минимальный
        Более простой документ: Calibri 11 пт, стандартные поля, минимум декоративного оформления.

Размер страницы
    A4
        Основной вариант для печати и редактирования.

    A5
        Более книжный формат страницы.

    Letter
        Американский формат страницы.

empty-line
    Игнорировать
        Тег FB2 empty-line не создаёт дополнительного абзаца.

    Пустой абзац
        empty-line преобразуется в пустой абзац.

    Интервал
        empty-line добавляет увеличенный интервал перед следующим абзацем.

Использовать свой шрифт
    Позволяет вручную выбрать основной шрифт и размер текста независимо от выбранного профиля.

Шрифт
    Выпадающий список установленных в Windows шрифтов. Вертикальные CJK-шрифты вида @Font скрываются.

Размер
    Размер основного текста в пунктах.


4. Дополнительно
~~~~~~~~~~~~~~~~
Записывать свойства DOCX из описания FB2
    Заполняет свойства DOCX: название, автор, тема/жанры, язык, дата, аннотация и другие доступные данные.

Проверять структуру DOCX после экспорта
    Выполняет проверку DOCX-пакета после формирования: обязательные части, связи, изображения, сноски, колонтитулы и служебные XML-файлы.

Создавать диагностический отчёт TXT
    Создаёт рядом с DOCX файл вида:

        имя_файла_export_report.txt

    В отчёт попадают количество глав, изображений, сносок, таблиц, гиперссылок и предупреждения.

Добавлять название книги в верхний колонтитул
    В верхний колонтитул добавляется название книги.

Добавлять номера страниц
    В нижний колонтитул добавляется поле PAGE Word.

Не нумеровать титульную/оглавление
    Создаёт отдельную секцию Word для титульной страницы и оглавления без видимого номера страницы.

Начинать нумерацию с основного текста
    Основной текст начинается с номера страницы 1 после титульной страницы/оглавления.

Включать автоматические переносы
    Включает автоматические переносы Word. Особенно полезно при выравнивании текста по ширине.

Открывать DOCX после экспорта
    После успешного экспорта открывает созданный DOCX приложением по умолчанию.

Язык документа
    Русский (ru-RU)
        Устанавливает русский язык проверки орфографии и переносов.

    Английский (en-US)
        Устанавливает английский язык документа.

    Авто из FB2 (<lang>)
        Берёт значение из description/title-info/lang и преобразует короткие коды ru, en, de, fr, uk, be, es, it, pl, cs, tr, zh, ja в подходящие языковые коды Word.

По умолчанию
    Восстанавливает безопасные настройки в текущем окне. Кнопка не закрывает окно; настройки сохраняются только после нажатия ОК.


Массовый экспорт
----------------
Для массовой обработки используется ExportDOCXBatch.exe. Удобнее запускать через обёртку:

    EXPORT_FOLDER.cmd "D:\Books" "D:\Books_DOCX"

Справка самой EXE-утилиты выводится на английском языке:

    ExportDOCXBatch.exe -Help
    ExportDOCXBatch.exe /?

Рекурсивно:

    EXPORT_FOLDER.cmd "D:\Books" "D:\Books_DOCX" -Recurse

Продолжить ранее прерванный экспорт:

    EXPORT_FOLDER.cmd "D:\Books" "D:\Books_DOCX" -Recurse -Resume

Проверить план без создания DOCX:

    EXPORT_FOLDER.cmd "D:\Books" "D:\Books_DOCX" -Recurse -DryRun

Экспорт по маске:

    EXPORT_FOLDER.cmd "D:\Books\*.fb2" "D:\Books_DOCX"

Экспорт по TXT-списку:

    EXPORT_FOLDER.cmd "-" "D:\Books_DOCX" -List "D:\lists\books.txt"

В TXT-списке можно указывать отдельные FB2, папки и маски. Пустые строки и строки, начинающиеся с # или ;, игнорируются.


Параметры ExportDOCXBatch.exe
-----------------------------
-Recurse
    Искать FB2 во вложенных папках.

-NoOverwrite
    Не перезаписывать уже существующие DOCX.

-Resume
    Продолжить экспорт: включает режим без перезаписи и дописывание лога.

-SkipIfNewer
    Пропускать DOCX, если он новее или ровесник исходного FB2.

-DryRun
    Только показать, что будет обработано, без создания DOCX.

-Flat
    При рекурсивном экспорте складывать все DOCX в одну выходную папку.

-Quiet
    Уменьшить вывод в консоль.

-OpenLog
    Открыть CSV-лог после завершения.

-OpenHtmlReport / -OpenHtml
    Открыть HTML-отчёт после завершения. Работает, если указан -HtmlReport path.

-OpenSummary
    Открыть TXT-сводку после завершения. Работает, если указан -Summary path.

-OpenFailedList
    Открыть TXT-список ошибок после завершения. Работает, если указан -FailedList path.

-OpenOkList
    Открыть TXT-список успешных файлов после завершения. Работает, если указан -OkList path.

-OpenReports
    Включить открытие всех созданных отчётов и списков: CSV, HTML, Summary, FailedList, OkList.

-PrintFailed
    В конце обработки вывести список FB2-файлов, завершившихся ошибкой, прямо в консоль.

-AppendLog
    Дописывать в существующий CSV-лог.

-StopOnError
    Остановиться после первой ошибки.

-Retries N
    Повторить неудачный экспорт N раз.

-ProgressEvery N
    Выводить прогресс каждые N файлов даже в тихом режиме.

-PreserveDates
    Перенести дату/время исходного FB2 на созданный DOCX.

-UniqueNames
    Подбирать уникальные имена DOCX, если выходной файл уже существует.

-Limit N
    Обработать не более N файлов.

-List path
    Добавить входные файлы из TXT-списка.

-FailedList path
    Записать список FB2, которые завершились ошибкой.

-OkList path
    Записать список успешно обработанных FB2.

-Summary path
    Создать краткий TXT-отчёт по массовой обработке.

-HtmlReport path
    Создать HTML-отчёт по массовой обработке.

-MinSize N / -MaxSize N
    Фильтр по размеру FB2. Поддерживаются суффиксы KB, MB, GB.

-ModifiedAfter yyyy-mm-dd / -ModifiedBefore yyyy-mm-dd
    Фильтр по дате изменения FB2.

-Dll path
    Явно указать ExportDOCX.dll.

-Log path
    Явно указать путь к CSV-логу.


Примеры batch-обработки
-----------------------
Обычный рекурсивный экспорт:

    EXPORT_FOLDER.cmd "D:\Books" "D:\Books_DOCX" -Recurse

Продолжить экспорт и пропускать уже актуальные DOCX:

    EXPORT_FOLDER.cmd "D:\Books" "D:\Books_DOCX" -Recurse -Resume -SkipIfNewer

Экспорт с HTML-отчётом и TXT-сводкой:

    EXPORT_FOLDER.cmd "D:\Books" "D:\Books_DOCX" -Recurse -HtmlReport "D:\Books_DOCX\report.html" -Summary "D:\Books_DOCX\summary.txt"

То же самое с автоматическим открытием отчётов:

    EXPORT_FOLDER.cmd "D:\Books" "D:\Books_DOCX" -Recurse -HtmlReport "D:\Books_DOCX\report.html" -Summary "D:\Books_DOCX\summary.txt" -OpenReports

Экспорт с повторными попытками и списком ошибок:

    EXPORT_FOLDER.cmd "D:\Books" "D:\Books_DOCX" -Recurse -Retries 2 -FailedList "D:\Books_DOCX\failed.txt"

Повторно обработать только ошибки прошлого запуска:

    EXPORT_FOLDER.cmd "-" "D:\Books_DOCX" -List "D:\Books_DOCX\failed.txt" -Resume

Экспорт с фильтром по размеру и дате:

    EXPORT_FOLDER.cmd "D:\Books" "D:\Books_DOCX" -Recurse -MinSize 10KB -MaxSize 80MB -ModifiedAfter 2026-01-01


Диагностика и отчёты
--------------------
При включённой опции диагностического отчёта плагин создаёт TXT-файл рядом с DOCX.

В отчёт могут попадать:

- количество обработанных разделов;
- количество изображений;
- количество сносок и примечаний;
- количество таблиц;
- количество гиперссылок;
- предупреждения о битых ссылках;
- предупреждения о не найденных binary;
- предупреждения о неизвестных типах изображений;
- результаты проверки структуры DOCX.

Для batch-экспорта создаётся CSV-лог. При указании -HtmlReport дополнительно создаётся HTML-отчёт.

В HTML-отчёте пути Input и Output являются кликабельными ссылками на локальные файлы. В некоторых браузерах открытие локальных file-ссылок может быть ограничено настройками безопасности.


Типовые проблемы
----------------
Плагин не появился в FB Editor
    Проверьте, что собрана конфигурация Release | Win32 и DLL зарегистрирована через REGISTER_EXPORTDOCX.cmd.

Ошибка при регистрации DLL
    Запустите командную строку от имени администратора и повторите REGISTER_EXPORTDOCX.cmd.

DLL зарегистрирована, но FB Editor её не видит
    Проверьте, что использовался 32-битный regsvr32:

        C:\Windows\SysWOW64\regsvr32.exe

    Не используйте C:\Windows\System32\regsvr32.exe для 32-битного FB Editor.

Word просит восстановить DOCX
    Включите проверку структуры DOCX и диагностический отчёт. После экспорта посмотрите файл *_export_report.txt.

Оглавление не обновлено
    В Word выделите оглавление и нажмите F9 либо выберите обновление полей.

Нет изображений
    Проверьте, что включены опции «Экспортировать изображения» и «Экспортировать обложку», а в FB2 есть соответствующие binary-блоки.

Неверный шрифт
    Включите «Использовать свой шрифт» и выберите шрифт из выпадающего списка установленных шрифтов Windows.

## Примечание по PVS-Studio

Часть предупреждений PVS-Studio по ATL/WTL-макросам `BEGIN_COM_MAP`, `MESSAGE_HANDLER` и COM-функции `DllGetClassObject` является ожидаемой: эти места должны сохранять сигнатуры COM ABI и структуру ATL/WTL-макросов. Для них в исходниках оставлены локальные комментарии подавления и пояснения.





Дополнение по CREATE_EXPORTDOCX_TEST_PACKAGE.cmd

--------------------------------------------

Командный файл CREATE_EXPORTDOCX_TEST_PACKAGE.cmd предназначен для подготовки архива,

который удобно отправлять на анализ. В начале файла задаются переменные SOURCE_DIR,

OUTPUT_DIR и PACKAGE_ZIP.



Важные параметры:

- INCLUDE_FB2_ZIP=1 — дополнительно распаковывает *.fb2.zip во временную staging-папку

  и конвертирует извлечённые FB2 вместе с обычными *.fb2;

- EXIT_WITH_EXPORT_CODE=0 — пакет создаётся с кодом 0 даже при ожидаемых FAIL в negative_tests;

- BATCH_ARGS — дополнительные параметры ExportDOCXBatch.exe.



В архив для анализа попадают:

- source_fb2\ — исходные FB2 и FB2 ZIP;

- converted_docx\ — созданные DOCX;

- logs\ — CSV/HTML/TXT отчёты, консольный лог, списки OK/FAIL, дерево исходной и staging-папки.



Консольный лог ExportDOCX_batch_console.log сохраняется в UTF-8.



### Crash protection in batch tests

`ExportDOCXBatch.exe` protects every single call to `ExportFB2FileToDOCX()` with SEH. If a malformed FB2 or an unexpected DLL error raises a structured exception, the current file is written to the CSV/HTML reports as `FAIL`, and the batch run continues with the next file.

The exported DLL entry point releases MSXML/COM objects before `CoUninitialize()`, which avoids access violations at function exit during batch conversion.

Notes and hyperlinks
--------------------
Only FB2 links with type="note" or links whose target section is inside body name="notes" are converted to Word footnotes/endnotes. Ordinary internal links, image links, appendix links and external URLs are kept as hyperlinks and are not counted as missing notes.

If a note contains images or external hyperlinks, the plugin creates a separate relationships part for footnotes.xml/endnotes.xml. This prevents Word from recovering the DOCX because of unresolved namespaces or relationships inside notes.


Дополнительно по внутренним ссылкам:
- ссылки на главы, абзацы, изображения, таблицы и другие элементы с id преобразуются в гиперссылки Word на закладки;
- ссылки type="note" не считаются примечаниями автоматически, если их цель находится в основном тексте, а не в body name="notes";
- это защищает от ложных сносок в FB2, где оглавление ошибочно размечено как type="note".


Дополнение: проверка DOCX и расширенная диагностика
----------------------------------------------------

В ExportDOCXBatch.exe добавлен параметр:

    -ValidateDocx

При его включении после успешного экспорта выполняется расширенная проверка созданного DOCX-пакета:

- файл должен быть корректным ZIP-контейнером;
- должны присутствовать обязательные части [Content_Types].xml, _rels/.rels, word/document.xml, word/_rels/document.xml.rels, word/styles.xml и word/settings.xml;
- проверяются пустые/битые XML и RELS-части;
- проверяются relationship Target внутри всех .rels-файлов;
- проверяются r:id, r:embed и r:link внутри XML-частей;
- проверяется наличие word/_rels/footnotes.xml.rels и word/_rels/endnotes.xml.rels при наличии footnotes.xml/endnotes.xml;
- проверяется наличие media-файлов и базовых типов изображений в [Content_Types].xml.

В CSV/HTML-отчётах появился дополнительный столбец Validation. Возможные статусы batch-обработки:

- OK — экспорт выполнен, базовая проверка DOCX не выявила проблем;
- OK_WITH_WARNINGS — DOCX создан, но в диагностическом отчёте файла есть предупреждения;
- DOCX_INVALID — DOCX создан, но базовая проверка структуры пакета выявила проблему;
- FAIL — экспорт не выполнен или функция экспорта вернула ошибку;
- SKIP, UPTODATE, FILTERED, DRYRUN — служебные статусы batch-режимов.

CREATE_EXPORTDOCX_TEST_PACKAGE.cmd теперь по умолчанию запускает batch-утилиту с -ValidateDocx.

Дополнительно рядом с CSV создаётся общий файл:

    warnings_summary.txt

В него сводятся предупреждения из отдельных *_export_report.txt, ошибки DOCX_INVALID, а также Expected FAIL / Unexpected FAIL. Отрицательные тесты из папки negative_tests учитываются как Expected FAIL, чтобы отличать ожидаемую проверку битого FB2 от настоящей регрессии.

Дополнительно диагностический TXT-отчёт по каждому файлу теперь содержит больше сведений о качестве конвертации:

- количество FB2 stylesheet;
- количество маленьких изображений, оставленных без растягивания;
- количество изображений, ограниченных по высоте страницы;
- статистику внутренних ссылок: всего, разрешено, битые цели, найденные id-цели и пропущенные дубликаты закладок.

Метаданные DOCX также расширены: кроме core.xml/app.xml создаётся docProps/custom.xml с FB2 ID, версией, program-used, жанрами, серией, src-title-info и publish-info. При включённом блоке сведений FB2 на титульной странице дополнительно выводятся publish-info, src-title-info и document-info/history.

Примечание: штатный формат FB2 не содержит MathML как поддерживаемый элемент. Если в исходном файле встретятся нестандартные XML-вставки, они не рассматриваются как отдельная функция экспорта; валидный FB2 экспортируется по поддерживаемым элементам формата, а формулы обычно сохраняются как текст или как изображения, если они уже оформлены в FB2 через <image>.

Изображения дополнительно ограничиваются по высоте рабочей области страницы, чтобы очень высокие картинки не выходили за пределы листа. Маленькие изображения не растягиваются.


Дополнительно: качество конвертации
-----------------------------------
В окне настроек добавлены пресеты экспорта:
- Книжный DOCX — сбалансированный вариант для чтения и печати: обложка, титульная страница, оглавление, стили FB2, ссылки, сноски, метаданные и нумерация страниц без лишних технических блоков.
- Минимальный DOCX — простой документ: основной текст и изображения, без обложки, титульной страницы, оглавления, гиперссылок, закладок, расширенных стилей, отчёта и нумерации страниц.
- Редакторский DOCX — вариант для проверки и доработки: всё из книжного режима плюс src-title-info, сведения FB2, history, колонтитулы, закладки, отчёты и расширенная проверка DOCX.

В окне настроек пресеты вынесены не в выпадающий список, а в три отдельные кнопки. Нажатие на кнопку сразу применяет соответствующий набор параметров; подробное описание доступно во всплывающей подсказке кнопки.

Улучшены: обложка, титульная страница, annotation/history/publish-info/src-title-info, стихи, эпиграфы, цитаты, подписи, таблицы, сноски, Heading 1..6, TOC 1..6 и обработка empty-line.

Стабилизация для большого объёма FB2
------------------------------------
- Добавлены RUN_LARGE_FB2_TEST.cmd и RUN_LARGE_FB2_DRYRUN.cmd.
- Добавлен ANALYZE_LARGE_FB2_RESULTS.cmd для анализа CSV после полного прогона.
- Добавлен README_LARGE_FB2_TEST.txt с рекомендуемым порядком тестирования.
- ExportDOCXBatch во время длинного запуска пишет маркеры текущего файла:
  ExportDOCX_current_file.txt и ExportDOCX_progress_state.txt рядом с CSV-логом.
- Для ночных прогонов рекомендуется использовать -Resume -SkipIfNewer -Retries 2
  -PreserveDates -ValidateDocx -ProgressEvery 25.
