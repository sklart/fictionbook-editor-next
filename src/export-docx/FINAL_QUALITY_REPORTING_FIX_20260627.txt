ExportDOCX final quality/reporting fix, 2026-06-27

Changes:
1. Images with content-type="application/octet-stream" are now identified by magic bytes after base64 decoding.
   Supported signatures: JPEG, PNG, GIF, BMP, TIFF.
   If the signature is recognized, the media part gets a proper extension/content type and no noisy unknown-content-type warning is emitted.

2. Batch summary/reporting is clarified:
   - SUCCESS_TOTAL = OK_CLEAN + OK_WITH_WARNINGS
   - OK_CLEAN = successful exports without warnings
   - OK_WITH_WARNINGS = successful exports with non-fatal warnings
   The same labels are written to summary.txt, HTML report, current file marker and progress state.

3. New warnings_files.txt report:
   - Default path: next to the CSV log.
   - CLI option: -WarningsFilesList <path> / -WarningsList <path>.
   - RUN_LARGE_FB2_TEST.cmd now passes this path explicitly.

4. Additional footnote hardening found during 01_OK_WITH_WARNINGS QA:
   Word footnotes/endnotes no longer contain nested w:footnoteReference elements.
   Note-to-note links inside note bodies are exported as plain visible text.
   This avoids problematic structures in documents like 18907.docx and prevents note cycles from leaking into OOXML footnotes.
