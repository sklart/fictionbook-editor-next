# Итоги ручного тестирования FBE от 2026-06-29

Источник: `docs/manual-test-plan.md` после ручного прогона и свежий `D:\FBE.plog`.

## Общий статус

Ручной smoke/regression-прогон в основном пройден: базовый запуск, открытие FB2, редактирование, структура документа, ссылки/сноски, изображения, поиск/замена, скрипты, основные экспорты, ImportEPUB, FBV, shell-интеграция и installer/uninstaller в пользовательских сценариях в целом работают.

Итоговый статус: **пройдено с замечаниями**.

## Блокеры / важные дефекты

1. FBE не давал понятного пользовательского сигнала при сохранении read-only FB2.
   - Статус: **закрыто консервативной правкой**.
   - Риск: потеря времени и правок, если пользователь не заметил, что файл был только для чтения.
   - Исправление: `SaveFile(false)` теперь заранее проверяет `FILE_ATTRIBUTE_READONLY`, показывает понятный диалог и предлагает сохранить копию через `Сохранить как...`; при отказе сохранение отменяется без снятия dirty-state.
   - Не делалось намеренно: автоматическое снятие атрибута read-only, чтобы не менять права/атрибуты файла без явного отдельного решения пользователя.

1. `create-release.ps1` / `verify-artifacts.ps1`: проверка symbols-архива падала.
   - Статус: **закрыто**.
   - Причина: `create-release.ps1` уже добавлял `ExportDOCX.pdb`, `ExportEPUB.pdb` и `ImportEPUB.pdb`, а `verify-artifacts.ps1` ожидал старый набор PDB.
   - Исправление: список expected symbols в `verify-artifacts.ps1` синхронизирован с release-пакетом; проверка текущего `out/artifacts` проходит.

2. ExportEPUB: при экспорте появляются окна с кракозябрами.
   - Статус: **исправлено в кодировке сборки, нужен короткий ручной GUI-smoke после установки**.
   - Приоритет: высокий для UX экспортного плагина.
   - Причина: C++-исходники ExportEPUB сохранены в UTF-8, но проект компилировался без `/utf-8`, из-за чего широкие строковые литералы интерпретировались через ACP.
   - Исправление: `ExportEPUB.vcxproj` и `ExportEPUBBatch.vcxproj` теперь собираются с `/utf-8`.

3. ExportEPUB: в Sigil в XHTML/OPF видны mojibake в `<title>`.
   - Статус: **закрыто автоматическим регрессионным тестом**.
   - Пример: `РћР±Р»РѕР¶РєР°`, `РђРЅРЅРѕС‚Р°С†РёСЏ`.
   - Приоритет: высокий для качества EPUB.
   - Исправление: после `/utf-8` экспорт кириллической FB2 проверяется тестом `tools/tests/test-export-epub-cyrillic.ps1`; он контролирует `content.opf`, `nav.xhtml`, `toc.ncx`, XHTML-главу и отсутствие типичных mojibake-маркеров.

4. ImportEPUB: обложка импортируется как встроенная, но не отображается в редакторе / не встроена в код ожидаемым образом.
   - Приоритет: средний-высокий.
   - Следующий шаг: проверить формирование `coverpage`, `binary` и ссылку `href` после импорта.

## Закрыто после отчёта

- Release-блокер по symbols ZIP закрыт: `verify-artifacts.ps1` синхронизирован с PDB новых плагинов, текущие артефакты проходят проверку.
- Read-only FB2 при обычном сохранении теперь обрабатывается явно: пользователь получает предупреждение и безопасный переход к `Сохранить как...`.
- ExportEPUB больше не портит кириллицу в XHTML/NCX `<title>` и навигации EPUB; исправление закреплено в `verify-release.ps1` через отдельный smoke-тест кириллического экспорта.

## Некритичные дефекты / UX-замечания

1. В настройках на разных вкладках используются разные шрифты/размеры.
2. `Save image as` для GIF предлагает JPEG, хотя на диск сохраняется GIF.
3. ExportDOCX: при экспорте книги с изображениями появляются предупреждения — требуется уточнить текст предупреждений и сценарий.
4. В окне `Параметры экспорта EPUB` пропали всплывающие подсказки к настройкам.
5. FBV при прямом запуске сразу скрывается: нужен режим выбора файлов/папки, а не только CLI/shell-сценарий.
6. Окно FBV пока не локализовано.
