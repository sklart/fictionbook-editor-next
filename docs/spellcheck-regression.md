# Регрессия орфографии

Короткий ручной чек-лист перед релизом или после обновления `Hunspell`,
словарей и связанного кода орфографии.

## Когда запускать

- после обновления `third_party/hunspell`;
- после замены файлов в `out/Release/dict`;
- после изменений в `src/fbe/Speller.cpp` или `src/fbe/Speller.h`;
- перед публикацией нового release-кандидата.

## Автоматические проверки

Сначала прогоняем встроенные проверки:

```powershell
.\tools\build\verify-release.ps1
```

Эта команда уже включает:

- проверку runtime-бинарников;
- проверку кодировок словарей;
- проверку того, что FBE получает кодировку словаря через `Hunspell`.

## Ручная проверка

Использовать [FBE.exe](D:\Download\FBeditor\out\Release\FBE.exe).

Подготовить короткий тестовый документ, где есть:

- английское слово с ошибкой, например `helo`;
- русское слово с ошибкой, например `превет`;
- немецкое слово с ошибкой, если проверяем `de_DE`;
- текст в режиме `Дизайн`;
- тот же текст в режиме `Код`.

Проверить по порядку:

1. Приложение запускается без аварии.
2. `Сервис -> Орфография` открывается без падения.
3. Для ошибочного русского слова окно орфографии показывает слово и список замен.
4. Для ошибочного английского слова окно орфографии показывает слово и список замен.
5. Если документ/фрагмент немецкий, для ошибочного немецкого слова тоже есть подсказки.
6. Кнопка `Игнорировать` переводит проверку к следующему слову без зависания.
7. Кнопка `Заменить` действительно меняет слово.
8. Кнопка `Добавить` не вызывает падение.
9. Проверка доходит до конца документа и завершает работу штатно.
10. В режиме `Код` поиск/замена и орфография не вызывают падения после открытия окна проверки.
11. Контекстное меню орфографии на слове с ошибкой не вызывает падения.
12. После завершения проверки редактор остаётся рабочим, выделение и фокус не ломаются.

## Что считать регрессией

Сразу останавливаем выпуск и разбираемся, если происходит хотя бы одно из
следующего:

- падение при открытии окна `Орфография`;
- пустое окно орфографии с последующим закрытием приложения;
- падение при показе подсказок замены;
- отсутствие подсказок для словаря, который точно есть в `out/Release/dict`;
- крах только на одном языке после обновления словарей;
- разные результаты между режимами `Дизайн` и `Код`, если такого отличия не ожидается.

## Если проблема повторилась

1. Повторить сценарий под `Visual Studio` через `Debug -> Start Debugging`.
2. Зафиксировать верхние строки `Call Stack`.
3. Проверить `%LOCALAPPDATA%\FBE\Crashes`.
4. Сверить `SET` в `.aff` словаря и логику определения кодировки в `Speller.cpp`.
